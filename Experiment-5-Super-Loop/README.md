# Experiment 5: Super Loop Based Embedded System using STM32

## Aim
To implement a super loop based embedded program that performs LED blinking, button reading, and IR sensor data acquisition using software timing.

---

## Components Required
- STM32 Nucleo-F446RE  
- Infrared Sensor  
- USB Cable  
- STM32CubeIDE  
- STM32CubeMX  

---

## Theory
A super loop is a simple embedded system architecture where all tasks are executed inside an infinite loop.

Instead of using blocking delays, the system uses a time base generated by SysTick timer. Software counters are used to track time intervals for different tasks.

Each task runs when its corresponding counter reaches a predefined value. This allows multiple tasks to run cooperatively within a single loop.

This method avoids the complexity of RTOS while still enabling multitasking behavior.

---

## Procedure
1. Open STM32CubeMX  
2. Select STM32F446RE  
3. Configure:
   - PA5 as LED output  
   - PC13 as button input  
   - PA0 as ADC input  
4. Generate code  
5. Open project in STM32CubeIDE  
6. Implement super loop logic  
7. Build and run  

---

## Code
while (1)
{
  uint32_t now = HAL_GetTick();
  uint32_t dt = now - last_time;
  last_time = now;

  led_counter += dt;
  button_counter += dt;
  sensor_counter += dt;

  if (led_counter >= 500)
  {
    led_counter = 0;
    HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5);
  }

  if (button_counter >= 50)
  {
    button_counter = 0;
    if (HAL_GPIO_ReadPin(GPIOC, GPIO_PIN_13) == 0)
    {
      HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET);
    }
  }

  if (sensor_counter >= 200)
  {
    sensor_counter = 0;
    HAL_ADC_Start(&hadc1);
    HAL_ADC_PollForConversion(&hadc1, 10);
    sensor_value = HAL_ADC_GetValue(&hadc1);
    HAL_ADC_Stop(&hadc1);
  }
}

## Result
LED blinking, button detection, and sensor reading were successfully performed using a single super loop.
